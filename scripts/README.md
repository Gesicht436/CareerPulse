# CareerPulse Scripts: Automation & Data Pipelines

The `scripts/` directory contains the operational backbone of the CareerPulse project. These scripts handle everything from data acquisition and database initialization to the generation of professional technical visuals and documentation conversion.

## Script Directory Overview

| Script | Purpose | Key Dependencies |
| :--- | :--- | :--- |
| `setup_data.py` | Automated dataset acquisition from Kaggle | `kaggle`, `python-dotenv` |
| `ingest_qdrant.py` | Vector embedding and Qdrant ingestion | `qdrant-client`, `pandas`, `sentence-transformers` |
| `generate_visuals.py` | Generation of architectural and performance diagrams | `matplotlib`, `numpy` |
| `md_to_docs.py` | Professional Markdown to DOCX converter | `pypandoc`, `pandoc` |

---

## 1. Data Acquisition: `setup_data.py`

This script is the first step in the CareerPulse pipeline. It automates the retrieval of large-scale job description datasets required to power the semantic search engine.

### How it Works

1. **Authentication**: It loads the `KAGGLE_API_TOKEN` from your `.env` file.
2. **Dataset Targeting**: It targets the `adityarajsrv/job-descriptions-2025-tech-and-non-tech-roles` dataset (or any other defined in the `DATASETS` list).
3. **Download & Extract**: It uses the Kaggle CLI API to download the dataset and automatically unzips it into the `core_engine/datasets/raw` directory.

### How to Run

```bash
# Ensure KAGGLE_API_TOKEN is in your .env
python scripts/setup_data.py
```

---

## 2. Vector Ingestion: `ingest_qdrant.py`

This is the heavy-lifter that populates the **Qdrant Vector Database**. It transforms raw text into high-dimensional mathematical vectors.

### How it Works

1. **Data Cleaning**: It reads the CSV downloaded by `setup_data.py` and cleans missing values.
2. **Semantic Synthesis**: It combines the Job Title, Skills, and Responsibilities into a single "Context String."
3. **Embedding Generation**: It passes the context string through the `EmbeddingService` (`all-MiniLM-L6-v2`), generating a **384-dimensional vector**.
4. **Payload Preparation**: It creates a metadata payload containing structured information (Title, Experience Level, Years of Experience, etc.).
5. **Batch Upsert**: To optimize network performance, it uploads the vectors to Qdrant in batches of 100 points.

### How to Run

```bash
# Ensure Qdrant is running (docker run -p 6333:6333 qdrant/qdrant)
python scripts/ingest_qdrant.py
```

## 3. Professional Visuals: `generate_visuals.py`

Documentation quality is a priority for CareerPulse. This script programmatically generates all the technical diagrams used in the project reports.

### How it Works

It uses `matplotlib` to create custom, professional-grade visuals with a consistent color palette. It generates:

- **System Architecture**: High-level component diagrams.
- **Security Flow**: Step-by-step visualization of the extraction and redaction pipeline.
- **RAG Workflow**: Diagram showing the Retrieval-Augmented Generation process.
- **Performance Graphs**: VRAM usage and Latency comparisons between different LLM models.
- **Mathematical Visuals**: Conceptualizing Cosine Similarity and Semantic Matching.

### How to Run

```bash
python scripts/generate_visuals.py
```

*Output: All images are saved to `project_assets/images/`.*

---

## 4. Documentation Converter: `md_to_docs.py`

A utility to bridge the gap between developer-friendly Markdown and professional stakeholder requirements (DOCX).

### How it Works

1. **Pandoc Integration**: It acts as a wrapper for the powerful `pandoc` engine via `pypandoc`.
2. **Resource Mapping**: It automatically identifies image paths relative to the Markdown file, ensuring diagrams generated by `generate_visuals.py` are correctly embedded in the DOCX.
3. **Format Handling**: It supports converting single files or entire directories recursively.

### How to Run

```bash
# Convert a single file
python scripts/md_to_docs.py README.md -o Final_Report.docx

# Convert an entire directory
python scripts/md_to_docs.py project_assets/
```

*Note: Requires Pandoc installed on the system (`winget install JohnMacFarlane.Pandoc`).*
